Java内部类

  1. 不允许同一个包中的其他类访问该类。
  2. 内部类成员可直接访问外部类的私有数据，反之不成立。
  3. 匿名内部类适合用于创建哪些仅需要一次使用的类。
  4. 成员内部类是一种和Field、方法、构造器相似的类内部。局部内部类和匿名内部类则不是类成员。
  5. 成员内部类分为两种：静态内部类和非静态内部类。
  6. 编译生成两个class文件，一个是外部类的class文件，一个是成员内部类的class文件OuterClass$InnerClass.class。

非静态内部类

非静态内部类和外部类成员的关系

非静态内部类里可以直接访问外部类的private成员，这是因为在非静态内部类对象里，保存了一个它寄存的外部类对象的引用（当调用非静态内部类的实例方法时，必须有一个非静态内部类实例，而非静态内部类实例必须是寄存在外部类实例里）。

非静态内部类对象和外部类对象的关系

非静态内部类对象必须寄存在外部类对象里，而外部类对象则不必一定有非静态内部类对象寄存其中。因此外部类对象访问非静态内部类成员，可能非静态普通内部类对象根本不存在，而非静态内部类对象访问外部类成员时，外部类对象一定存在。

非静态内部类不允许定义静态成员。

##静态内部类
静态内部类属于外部类本身。（static关键字的作用是把类的成员变成类相关而不是实例相关）
静态内部类不能访问外部类的实例成员，只能访问外部类的类成员。
外部类不能直接访问静态内部类的成员，但可以使用静态内部类的类名作为调用者来访问静态内部类成员，也可以使用静态内部类对象作为调用者来访问静态内部类的实例成员。

Java基本类型与包装类的区别

####JVM的内存区域组成
java把内存非为栈内存，堆内存
(1)在函数中定义的基本类型变量和对象的引用变量都在函数的栈内存中分配；
(2)堆内存用来存放由new创建的对象和数组以及对象的实例变量。在函数（代码块）中定义一个变量时，java就在栈中为这个变量分配内存空间，当超过变量的作用域后，java会自动释放掉为该变量所分配的内存空间；在堆中分配的内存由java虚拟机的自动垃圾回收器来管理

java中数据在内存中是如何存储的

  1. 基本数据类型 java的基本数据类型有8种，即int,short,long,byte,float,double,boolean,char(注意，并没有String的基本类型)。这种类型的定义是通过诸如int a = 3；long b = 255L；的形式来定义的。如int a = 3；这里的a是一个指向int类型的引用，指向3这个字面值。这些字面值的数据，由于大小可知，生存期可知(这些字面值定义在某个程序块里面，程序块退出后，字段值就消失了)，出于追求速度的原因，就存在于栈中。
    栈有一个很重要的特殊性，就是存在栈中的数据可以共享。
  2. 对象
    在java中，创建一个对象不包括对象的声明和实例化。
    (1)声明对象时，将在栈内存为对象的引用变量分配内存空间，但此时对象是一个空对象，不能使用，因为它还没有引用任何实体。
    (2)对象实例化时，堆内存会为类的成员变量分配内存，将其初始化为各数据类型的默认值；接着进行进行显式初始化；最后调用构造方法，为成员变量赋值。返回堆内存中的对象。
    (3)创建多个不同的对象实例
    一个类通过使用new运算符可以创建多个不同的对象实例，这些对象实例将在堆中被分配不同的内存空间。在堆内存中分别为不同对象的成员变量分配内存空间。
    (4)包装类
    基本类型都有对应的包装类。基本类型的定义都是直接在栈中，如果用包装类来创建对象，就和普通对象一样了。例如：int i=0；i直接存储在栈中。Integer i（i此时是对象）= new Integer(5)；这样，i对象数据存储在堆中，i的引用存储在栈中，通过栈中的引用来操作对象。
    (5)String
    String是一个引用类型。

javascript引用类型

引用类型是一种数据结构，用于将数据和功能组织在一起，与类相似。对象是某个特定引用类型的实例。新对象是使用new操作符后跟一个构造函数来创建。ECMAScript提供了很多原生引用类型。

Object类型

创建Object实例的方式有两种：

  1. new操作符后跟Object构造函数
    var person = new Object();
  2. 使用对象字面量表示法，目的在于简化创建包含大量属性的对象的过程
    var person = {
    name : “Becky”,
    age : 22,
    sex : 女
    };
    对象字面量也可封装函数参数
Array类型

数组的项数保存在其length属性中，这个属性并非只读
创建数组的基本方式有两种：

  1. 使用Array构造函数
    var colors = new Array();
    如果预先知道数组的长度，var colors = new Array(10);

  2. 使用数组字面量表示法
    var colors = [“red”,”blue”,”white”];
    colors[3] = “black”; //新增第四项

javascript垃圾收集

首先要对内存管理概念有一个基本的认识，内存管理是指操作系统如何进行内存的分配和回收的机制。C和C++通过malloc，free函数来向操作系统请求内存和释放内存。这种机制的优点是分配内存和释放的效率很高，缺点是存在大量的内存分配和释放操作，程序员不小心忘记释放内存会造成内存泄漏。对于长期运行的软件来讲，这将是一个致命的威胁，因为系统的内存会逐渐被吃光。

因此，更新的编程语言，比如JAVA,C#,都提供了所谓“垃圾回收的机制”，运行时自身会运行相应的垃圾回收机制。程序员只需要申请内存，而不需要关注内存的释放。垃圾回收器(GC)会在适当的时候将已经终止生命周期的变量的内存给释放掉。

  javascript具有自动垃圾收集机制，执行环境会负责管理代码执行过程中使用的内存。

垃圾收集器标识无用变量的策略一般有两个：标记清除（常见）和引用计数。

javascript在进行内存管理及垃圾收集时最主要的一个问题就是分配给Web浏览器的可用内存数量通常通常比分给桌面应用程序的少（目的是防止运行JavaScript的网页耗尽全部系统内存而导致系统崩溃）。优化内存占用的最佳方式就是为执行中的代码只保存必要的数据，一旦数据不再有用，最好解除引用（通过将全局变量和全局对象的属性的值设置为null来释放其引用）

总结一下大致过程：

安装Hexo

1.任意位置Git Bash，输入命令“npm install -g hexo
2.创建Hexo文件夹后，进入，Git Bash输入“hexo init”
3.安装依赖包：“npm install”
4.执行“hexo generate”和“hexo server”即可在本地localhost：4000查看

创建Repository

注意名字是Yourname.github.io
1.修改配置文件_config.yml里的deploy，
“type: git
repo: https://github.com/becky76/becky76.github.io
branch: master”

设置SSH keys

配置SSH的官方教程是：https://help.github.com/articles/generating-ssh-keys/
1.先检查是否存在了SSH keys “ls -al ~/.ssh
2.存在的话直接删除.ssh文件夹里面所有文件，.ssh文件夹在密钥文件在C:\Users\n.ssh
3.输入以下指令生成新的ssh key“ssh-keygen -t rsa -C “xxx@163.com””（邮箱是注册Github时用的邮箱），提示输入passphrase时可以直接回车不输入。
4.启动ssh-agent “ssh-agent -s”
5.添加新ssh key到ssh-agent “ssh-add ~/.ssh/id_rsa”
6.成功后拷贝id_rsa.pub到settings->SSH keys添加ssh key
7.是否成功添加SSH key可通过“ssh -T git@github.com”查询

完成部署

1.“hexo generate”和“hexo deploy”，浏览器打开“http://becky76.github.io/”

写文章

1.hexo new “title”
2.编辑source_posts\article.md
注意每次修改本地文件后，需要在Hexo目录下键入“hexo generate”才能保存
